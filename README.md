## Функционал

### Асинхронная очередь (```AsyncTaskQueue```)
Очередь на ```deque``` с асинхронной синхронизацией через ```Condition```. Содержит следующие методы:
- ```put()``` — асинхронно добавляет задачу в очередь
- ```get()``` — асинхронно получает задачу из очереди (ожидает, если пусто)
- ```qsize()``` — возвращает текущий размер очереди
- ```empty()``` — проверяет, пуста ли очередь
- ```full()``` — Проверяет, заполнена ли очередь (если есть ограничение)
- ```clear()``` — очищает очередь

### Протокол обработчика (```TaskHandler```)
Контракт на ```Protocol``` с методом ```handle``` — асинхронно обрабатывает задачу.

### Обработчики задач (```EmailTaskHandler```, ```NotificationHandler```, ```PkgTaskHandler```)
Обработчики на асинхронных функциях с имитацией задержки.

### Асинхронный исполнитель (```AsyncTaskExecutor```)
Исполнитель с параллельной обработкой задач. Содержит следующие методы:
- ```register_handler()``` — регистрирует обработчик для типа задач
- ```run()``` — запускает исполнителя
- ```stop()``` — останавливает исполнителя

### Очередь задач (```TaskQueue```)
Очередь задач с поддержкой итерации и ленивой фильтрации. Содержит следующие методы:
- ```__init__``` — инициализируем пустую очередь
- ```add_task``` — добавляет задачу в очередь
- ```__len__``` — возвращает количество задач
- ```__getitem__``` — доступ к задаче по индексу
- ```__iter__``` — возвращает итератор для обхода
- ```filter_by_status``` — ленивая фильтрация по статусу
- ```filter_by_priority``` — ленивая фильтрация по диапазону приоритета
- ```clear``` — очищает очередь

### Итератор (```TaskQueueIterator```)
Отдельный класс итератора с обработкой StopIteration:
- ```__init__``` — инициализирует сам итератор
- ```__iter__``` — возвращает сам итератор
- ```__next__``` — возвращает следующую задачу или выбрасывает StopIteration
  
### Модель задачи (```Task```)
Модель задачи с валидацией через дескрипторы. Содержит следующие атрибуты:
- ```id``` (число) — уникальный номер задачи
- ```payload``` (словарь) — любые данные, которые нужно обработать
- ```description``` (строка) — описание задачи
- ```priority``` (число) — приоритет задачи ( от 1 до 10)
- ```status``` (строка) — Статус задачи (```created```, ```in_progress```, ```completed```, ```failed```)
- ```created_at``` (datetime) — Время создания

### Протокол источника задач (```TaskSource```)
Это договор, которому должны соответствовать все источники задач.

### Реализованные источники задач:
1) ```TaskSourceGenerator``` ( Генератор случайных задач)
   - Создаёт 10 задач со случайными данными
   - ```id```: случайное число от 1 до 100
   - ```payload```: {"user_id": случайное, "count": случайное}
   - ```description```: "Сгенерированная задача"
   - ```priority```: случайное число от 1 до 10
   - ```status```: случайный статус

2) ```TaskSourceFile``` ( Чтение из JSON-файла)
   - Читает задачи из файла
  
3) ```TaskSourceAPI``` ( Имитация внешнего API)
   - Возвращает 4 задачи с фиксированными ```id``` (10-13)
   - Данные генерируются случайно при каждом вызове
   - Имитирует сетевое взаимодействие
   - ```description```: "Задача из API"
   - ```status```: статус по умолчанию (```created```)

### Реализованные Дескрипторы:
1) ```IntegerValidator``` ( Для целочисленных полей)
   - Проверка типа (должен быть int)
   - Проверка минимального значения
   - Проверка максимального значения
2) ```StringValidator``` ( Для строковых полей)
   - Проверка типа (должен быть str)
   - Проверка на пустую строку
3) ```StatusValidator``` ( Для статуса задачи)
   - Проверка допустимых значений: created, in_progress, completed, failed
   
## Технические решения
- Класс ```Task``` реализуется через обычный класс
- Все источники реализуют метод ```get_tasks()``` ( Duck typing)
- Используется ```isinstance()``` для проверки соответствия протоколу (декоратор - ```runtime_checkable```)
- Единый контракт через typing.Protocol ( Классы ```TaskSourceGenerator```, ```TaskSourceFile```, ```TaskSourceAPI``` не наследуются от ```TaskSource```)
- Дескрипторы - реализованы классы ```IntegerValidator```, ```StringValidator```, ```StatusValidator``` с методами ```__init__```, ```__get__```, ```__set__```
- Вычисляемые свойства через @property (```created_at```, ```is_ready```, ```is_active```, ```is_completed```, ```is_failed```)
- Методы изменения статуса (```start```, ```cancel```, ```complete```)
- Пользовательские исключения - ```TaskStateError``` для ошибок статуса
- Проверка данных при установке атрибутов
- Явная обработка StopIteration при завершении обхода
- Экономия памяти при работе с большими объёмами данных
- Контекстный менеджер через ```__aenter__``` и ```__aexit__```
- Синхронизация через ```asyncio.Condition```
